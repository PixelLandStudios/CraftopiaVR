VR Console Log System Documentation

Introduction

The VR Console Log System enhances debugging in virtual reality environments by enabling developers to view Unity console logs directly within their VR headset. This system eliminates the need to remove the headset to check logs on a desktop, streamlining the debugging process and enhancing efficiency by displaying errors and logs in real time within the VR environment.

Integration Tips

To integrate the VR Console Log System into your VR project, simply drag and drop the provided prefab into your scene. The system offers adjustable settings for screen size, text size, and color preferences to accommodate various development needs.

The Meta Quest scenes utilize the Meta XR All-In-One SDK v64.0 and the Meta XR Interaction SDK OVR Samples v64.0.

The XR Toolkit scenes are built using the XR Interaction Toolkit 2.5.4 and the toolkit’s example scene.

Components Overview

LogManager

Purpose: Acts as the central repository for all log data, capturing Unity console logs and storing them. LogManager broadcasts updates when new logs are recorded, ensuring all connected components receive the latest log information.

VRConsoleLogger

Purpose: Displays log messages within the VR environment. It interfaces directly with LogManager to receive and display logs dynamically. The logger manages a pool of LogMessage prefabs for efficient display and supports interactions such as log filtering and scrolling. By default, toggling the logger simply hides it, but it can also be completely disabled and later re-enabled. When reactivated, the logger rebuilds the display using the most recent log entries up to the specified maximum message count. This feature is designed to manage performance by limiting the number of complex UI elements active at any time. Additionally, all inspector variables provide live feedback during adjustments outside of gameplay, facilitated by a custom editor script.

ButtonSequenceDetector

Purpose: Allows for a covert method of activating the VRConsoleLogger through a sequence of button presses known only to the developer. This feature provides secure and discreet access to log information, ensuring that debugging tools remain hidden from users or testers.

Description: The ButtonSequenceDetector script is designed to be flexible, accommodating projects that utilize either Unity’s traditional input system or the newer Input System package. By default, the script is set up to use Unity’s old input system using KeyCode. To use the new input system version, the script must be modified by commenting out the old input region and uncommenting the new input region. This is done to prevent errors from people not using the new input system.

LogMessage

Purpose: Manages the individual display of log entries. Each log type (error, warning, info) is formatted for easy identification and interaction, facilitating quicker debugging directly within the VR interface.

MoveToObject

Purpose: Dynamically adjusts the position of the VRConsoleLogger within the VR environment, optimizing visibility and accessibility based on the user’s current interaction context.

VRConsoleLoggerButtons

Purpose: Provides UI elements that enable developers to filter and control the display of log messages. This component enhances the debugging interface by allowing for real-time adjustments to what is displayed.

VRConsoleLogToggle

Purpose: Manages individual toggle controls for each log type, providing an intuitive interface for developers to quickly adjust log visibility according to their current needs.

DynamicComponentToggle

Purpose: The DynamicComponentToggle is a flexible utility class designed to manage the enabled state of multiple component types across a hierarchy of GameObjects in a Unity scene. This class provides an efficient means to dynamically enable or disable groups of components based on their type without needing a reference to a specific API. This only works by dragging scripts from within the scene.

Prefab Overview

The VR Console Log System comes with a set of default prefabs located in the “Prefabs” folder, which are essential for the implementation within the main example scenes. These prefabs are designed for intuitive integration and dynamic interaction within VR environments, ensuring a seamless debugging experience.

Key Prefab Types:

  • Basic Logger: A straightforward log display that can be placed statically anywhere within the scene. It is designed to respond to user interactions, allowing developers to interact with log data seamlessly.
  • Movable Logger: This logger can be initially placed on any surface, like a wall, and is designed to be movable. It can be grabbed and repositioned by the player, providing flexibility in how and where log data is viewed.
  • Advanced Logger: An advanced logger that automatically positions itself in front of the player when toggled on. This logger also features grab-and-move capabilities, allowing for personalized placement within the user’s field of view. The meta version also includes scaling and rotating.
  • Wrist Logger: This logger is designed to appear on the player’s left wrist and can be toggled on or off. It provides convenient access to log data without interfering with the primary gameplay or VR interaction.
